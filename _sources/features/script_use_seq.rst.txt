转换为序列工程
====================================

将单帧工程转换为序列工程。

在菜单中选择 :guilabel:`编辑` - :guilabel:`转换为序列工程` - :guilabel:`设置`  进行设置。

在菜单中选择 :guilabel:`编辑` - :guilabel:`转换为序列工程` - :guilabel:`对当前工程执行` 
对当前打开的工程进行处理。

在菜单中选择 :guilabel:`编辑` - :guilabel:`转换为序列工程` - :guilabel:`对文件夹执行`
对指定文件夹中所有的 .nk 文件进行处理。

素材替换
---------------------------

将文件中单帧素材替换为序列素材。

只有单帧素材文件名和发现文件中的某个文件完全相同匹配时才会进行替换，如有多个匹配序列，将使用首个匹配。

替换后节点的时间范围将自动设为新素材的节点范围。

由用户设置序列发现规则。

- `*` 代表任意个数的非文件夹分隔符( ``\`` 或 ``/`` )字符
- `**` 代表任意个数的字符

节点替换
---------------------

支持文件中的单个输出节点替换为 `wlf_Write`，如有多个输出节点将报错。

工程设置
--------------------------

支持覆盖工程目录设置

工程目录
~~~~~~~~~~~~~~~~~~~~~~~

可由用户指定覆盖工程目录，可设置为 ``[python {nuke.script_directory()}]`` 代表使用脚本保存目录作为工程目录。

指定空值代表保持原有工程目录。

时间范围
~~~~~~~~~~~~~~~~~~~~~

可将工程时间范围将设为由工程所有非单帧且未出错读取节点的时间范围合并值。


批量输出
----------------------

支持批量对一个文件夹中的所有.nk 文件进行转换，发现重名文件将报错。

所有转换完的 nk 文件将保存至到用户指定的文件夹。
